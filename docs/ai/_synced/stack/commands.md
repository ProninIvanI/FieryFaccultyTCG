# Команды стека web-ccg

Версия: 1.0  
Дата: 2026-03-15

## 1. Frontend

- `npm run dev` / `pnpm dev`:
  - запуск Vite dev‑сервера фронтенда;
  - используется для локальной разработки UI.
- `npm run build` / `pnpm build`:
  - production‑сборка фронтенда;
  - должна выполняться без ошибок и предупреждений.
- `npm run lint` / `pnpm lint`:
  - запуск ESLint для фронтенда;
  - обязателен перед коммитом или в рамках `analysis` стадии CI.

---

## 2. Backend и game-core

- `npm run dev:server`:
  - запуск сервера (Node.js + WebSocket) в режиме разработки.
- `npm run build:server`:
  - сборка серверного кода в production‑формат (например, через `tsc`).
- `npm run test`:
  - запуск unit‑/application‑тестов:
    - для `game-core`;
    - для серверной части.

---

## 3. Общие команды качества

- `npm run lint`:
  - может быть сконфигурирована для одновременной проверки frontend + backend;
  - ошибки линтера блокируют merge.
- `npm run typecheck`:
  - отдельная команда для `tsc --noEmit`;
  - используется как часть `analysis` стадии в CI.

---

## 4. Docker / docker-compose (если используются)

- `docker build ...`:
  - сборка production‑образа сервера;
  - образ не должен содержать dev‑зависимости.
- `docker-compose up`:
  - локальный запуск всего стенда (сервер, БД/брокер, если есть);
  - применяется для интеграционного тестирования и дебага.

---

## 5. Интеграция с justfile (опционально)

При использовании `just`:

- `just dev`:
  - может оборачивать запуск фронтенда и сервера (параллельно или через отдельные рецепты).
- `just check`:
  - объединяет `lint`, `typecheck` и тесты;
  - используется перед merge и в CI для стадии `analysis`/`test`.
