# Стиль кода стека web-ccg (frontend + backend)

Версия: 1.0  
Дата: 2026-03-15

## 1. Общие правила TypeScript

- Включён `strict` и другие строгие опции.
- Запрещены:
  - `any`, `as any`, `as unknown as T` без крайне веской причины;
  - злоупотребление оператором `!` (non-null assertion).
- Для внешних данных:
  - ввод из сети, файлов, `localStorage` описывается через отдельные типы DTO;
  - есть мапперы из DTO в доменные типы.

---

## 2. Стиль фронтенда (React 18 + Vite)

- Компоненты:
  - типизируются через явный тип пропсов:
    - `type Props = {...}`;
    - `function Component(props: Props)` или `const Component: React.FC<Props> = ...`;
  - не используют устаревший React‑API (class components, legacy lifecycle).
- Состояние:
  - по умолчанию — локальный `useState`/`useReducer` и подъем состояния вверх;
  - глобальное состояние (если появится) — минимальное, с чётко описанным контрактом.
- Импорты:
  - использовать алиасы (`@/shared`, `@/features` и т.п.), а не цепочки `../../../../`.
- UI:
  - компоненты максимально «глупые», без бизнес‑логики.

---

## 3. Стиль backend (Node.js + WebSocket)

- Модульность:
  - разделение по слоям: `transport/ws`, `application`, `domain/game`, `infrastructure`;
  - каждый модуль несёт одну ответственность.
- Контракты:
  - экспортируются явные типы команд, событий и state;
  - публичный API `game-core` описан и не зависит от инфраструктуры.
- Логирование:
  - использовать единый logger‑модуль;
  - уровни логов (`debug`, `info`, `warn`, `error`) выбираются осознанно.

---

## 4. Линтеры и форматирование

- ESLint:
  - правила включены и для фронтенда, и для бэкенда;
  - правила TS/React/Node настроены так, чтобы:
    - запрещать неиспользуемый код;
    - выявлять потенциальные ошибки типов и асинхронности.
- Форматирование:
  - единый форматтер (Prettier или аналог);
  - автоформатирование перед коммитом (pre-commit hook или отдельная команда).

---

## 5. Специфические для игры ограничения

- Никакая логика, влияющая на исход матча, не реализуется:
  - во фронтенде;
  - во вспомогательных слоях сервера вне `game-core`.
- Любые обоснованные отступления от этих правил:
  - документируются и сопровождаются комментариями, поясняющими причину.
