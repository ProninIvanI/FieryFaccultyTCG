# Git workflow и релизы

Версия: 1.0
Последнее обновление: 2026-02-20

Ветвление, merge requests, коммит-сообщения. Обязательны для всех проектов.

---

## Branch naming

| Префикс | Назначение |
|---------|------------|
| `feature/...` | Новая функциональность |
| `bugfix/...` | Исправление багов |
| `hotfix/...` | Критические исправления |
| `refactor/...` | Рефакторинг |

---

## Workflow

- Merge request в основную ветку.
- Обязательное code review при merge.
- Squash commits при merge для чистой истории.
- `git rebase` для обновления feature-веток.

---

## Коммит-сообщения

- На **английском** языке.
- Осмысленные и описательные.
- Atomic commits: один логический блок изменений.

---

## Pre-commit checks

- Перед merge request выполняется полный набор проверок, определённый в justfile проекта.
- Swagger/API-документация обновляется при изменении эндпоинтов.

---

## Запреты

- Не коммитить `.env` файлы.
- Не коммитить sensitive данные.
- Изменения фиксируются в `CHANGELOG.md`.
