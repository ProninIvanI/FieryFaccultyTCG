# Безопасность

Версия: 1.0
Последнее обновление: 2026-02-20

Правила безопасности кода и данных. Обязательны для всех проектов.

---

## SQL-инъекции

- Все SQL-запросы выполняются через подготовленные выражения или query builder с параметризацией.
- Прямая конкатенация строк в SQL запрещается.

---

## Секреты и .env

- `.env` файлы не коммитятся в репозиторий.
- Переменные окружения - единственный способ передачи конфигурации.
- В `env/` хранятся только примеры с placeholder-значениями.

---

## Запрет логирования sensitive данных

Запрещено выводить в логи:
- Пароли и токены
- Полные SQL-запросы с пользовательскими данными
- Значения из `.env` файлов
- Персональные данные пользователей (email, телефон, ФИО)

---

## Валидация

- Входные данные валидируются на уровне Handler.
- CORS middleware настроен.
- Авторизация проверяется в middleware.

---

## Graceful shutdown

- Обеспечивает корректное завершение при SIGINT/SIGTERM.
- Применяется к HTTP-серверу, consumers и producers.
