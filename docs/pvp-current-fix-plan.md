# PvP current fix plan

## Цель

Собрать актуальный список правок по PvP после smoke-проверки и вести его как рабочий документ: добавлять пункты, удалять неактуальные, отмечать выполненное и согласовывать порядок перед реализацией.

Правки делаем пакетами: сначала согласуем пункт, потом меняем код, затем выполняем кодовые проверки. Визуальный smoke после правок выполняется вручную пользователем.

## Текущий план

### 1. Вернуть нормальный card-shell для ленты

Статус: не начато.

Проблема:

- Карта в руке выглядит как нормальная единая карта.
- После выкладывания карты в ленту драфта элементы визуально распадаются: art, название, `SPD`, mana-флажок и кнопка удаления выглядят как отдельные части вне корпуса карты.
- Это регресс: раньше выложенная карта воспринималась как полноценная карта, а не как набор разрозненных элементов.

Что нужно сделать:

- Карта в ленте должна выглядеть как карта из руки: единый корпус, art, стоимость/скорость, название и управляющие элементы внутри рамки.
- Стабилизировать размеры карты, чтобы она не меняла высоту из-за действий или дополнительной информации.
- Кнопка удаления должна быть частью card-shell или компактным контролом, который не разрушает силуэт карты.

### 2. Развести карту и служебные действия

Статус: не начато.

Проблема:

- Действия существа сейчас лепятся прямо на карту и перекрывают важные части.
- Карта начинает выглядеть как панель управления, а не как игровая карта.

Что нужно сделать:

- Карта остаётся чистой и читаемой.
- Уже добавленные действия показывать через маленький флажок/ярлык на карте.
- При наведении на флажок раскрывать компактный popover/“свиток” со списком действий: атака, уклонение, цель, ошибки.
- Действия не должны менять высоту карты и не должны растягивать layout.

### 3. Переделать кнопки действий существа

Статус: не начато.

Проблема:

- Текущие CSS-иконки атаки/уклонения выглядят нечётко и плохо читаются.
- Кнопки действий находятся слишком близко к содержимому карты и визуально шумят.

Что нужно сделать:

- Заменить мутные CSS-иконки на чёткие понятные пиктограммы.
- Сделать компактную action-toolbar для выбранного существа.
- Разместить toolbar так, чтобы он не перекрывал название, art и статы карты.
- У каждой кнопки должны быть нормальные `aria-label` и `title`.

### 4. Вернуть понятные target-флажки на поле

Статус: не начато.

Проблема:

- При выборе карты/действия не все допустимые цели выглядят доступными.
- Флажок цели должен показывать сущности, которые можно атаковать или выбрать целью, но сейчас это поведение неочевидно.

Что нужно сделать:

- Если выбрана карта или действие, все допустимые цели должны явно подсвечиваться.
- На вражеском маге и на призванных существах должны появляться понятные маркеры “можно выбрать целью”.
- Клик по NPC должен менять цель выбранной карты/действия так же, как клик по магу.
- Активная выбранная цель должна визуально отличаться от просто доступной цели.

### 5. Разобраться с `targetType` боевых карт

Статус: выполнено для direct single-target damage spells (2026-05-06).

Проблема:

- У части прямых damage-заклинаний в данных стоит `targetType: "enemyCharacter"`, поэтому они могут выбирать только мага.
- Игрок ожидает, что одиночное атакующее заклинание сможет выбрать либо мага, либо призванное существо, если существо стоит на поле.
- Во фронте уже есть поддержка `enemyAny`, где кандидатами становятся и маг, и вражеские существа.

Что нужно решить:

- Либо все одиночные прямые damage-заклинания должны получить `targetType: "enemyAny"`.
- Либо часть карт сознательно бьёт только мага, но тогда это должно быть явно отражено в описании и визуале.

Предварительное предложение:

- Прямые single-target damage spells перевести на `enemyAny`.
- Массовые эффекты, ловушки, глобальные debuff/control и специальные карты разбирать отдельно по смыслу эффекта.

Результат:

- Direct single-target `spellKind: "damage"` карты переведены на `targetType: "enemyAny"`.
- Массовые эффекты, ловушки, DoT/displacement и специальные карты оставлены без изменения для отдельного разбора.
- `game-core` валидация реальной damage-карты против вражеского существа покрыта тестом.

### 6. Обновить тесты

Статус: частично выполнено для пакета `targetType`/выбор целей (2026-05-06).

Что нужно покрыть:

- `game-core`: реальные damage cards, которые должны бить NPC, имеют корректный `targetType` и проходят валидацию против вражеского существа.
- `frontend`: карта с `enemyAny` показывает и мага, и вражеское существо как доступные цели.
- `frontend`: выбор NPC обновляет цель action-card и inspect/summary.
- Существующие сценарии для `enemyCharacter` оставить там, где карта действительно должна бить только мага.

Результат по текущему пакету:

- Добавлен `game-core` regression test на реальные direct single-target damage cards и валидацию цели-существа.
- Обновлены `frontend` ожидания для direct damage card с `enemyAny`.
- Существующие `enemyCharacter` сценарии для атаки существ сохранены.

### 7. Проверки после правок

Статус: выполнено для пакета `targetType`/выбор целей (2026-05-06).

Минимальные проверки:

- `npm.cmd --prefix frontend run type-check`
- `npm.cmd --prefix frontend run lint`
- `npm.cmd --prefix frontend run build`

Если меняем PvP UI:

- `npm.cmd --prefix frontend run test -- PlayPvpPage.test.tsx`

Если меняем карточные данные, target validation или эффекты:

- `npm.cmd --prefix game-core run test`

Результат по текущему пакету:

- `npm.cmd --prefix frontend run lint` — пройдено.
- `npm.cmd --prefix frontend run build` — пройдено.
- `npm.cmd --prefix frontend run test -- PlayPvpPage.test.tsx` — пройдено.

## Предлагаемый порядок реализации

1. Сначала исправить `targetType` и выбор целей, потому что это геймплейный баг.
2. Затем привести card-shell ленты к нормальному виду карты.
3. Затем вынести действия существа в флажок/popover и заменить мутные кнопки на понятные иконки.
4. После каждого пакета выполнять кодовые проверки и отдавать пользователю на ручной visual smoke.
